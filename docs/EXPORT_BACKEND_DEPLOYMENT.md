# Export Backend Deployment

The production frontend is static. Professional PDFs and office-ready Excel workbooks are generated by the FastAPI service in `backend/`.

## Required Endpoints

- `GET /health`
- `POST /exports/quotation`
- `POST /exports/invoice`
- `POST /exports/receipt`
- `POST /exports/client-statement`
- `POST /exports/excel`

Compatibility endpoints are also available for older callers:

- `POST /api/export/pdf`
- `POST /api/export/excel`
- `POST /export/pdf`
- `POST /export/excel`

## Environment

Render or Docker should provide:

```text
PORT=8765
CGM_EXPORT_ENVIRONMENT=production
CGM_EXPORT_ALLOWED_ORIGINS=https://your-vercel-app.vercel.app
CGM_EXPORT_FRONTEND_ORIGIN=https://your-vercel-app.vercel.app
CGM_EXPORT_ADDITIONAL_ALLOWED_ORIGINS=https://other-approved-origin.example
CGM_EXPORT_LOGO_PATH=assets/logo.png
CGM_EXPORT_DEFAULT_CURRENCY=BWP
```

The frontend must use:

```text
VITE_EXPORT_API_BASE_URL=https://your-render-export-service.onrender.com
```

## Local Run

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

## Production QA

Before office use, confirm:

- `/health` returns status `ok`.
- quotation, invoice, receipt, and client statement payloads return `%PDF` files.
- Excel reports return `.xlsx` files.
- known document types never return generic field/value PDFs.
- CORS allows the deployed Vercel origin only.
