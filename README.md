# Civil-Gineer Masta Accounting App

Production office system for Civil-Gineer Masta Proprietary Limited covering clients, projects, quotations, invoices, receipts, expenses, supplier bills, reports, backups, and audit records.

## Frontend

```bash
npm install
npm run dev
npm run build
```

Required production variables:

```text
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-supabase-anon-or-publishable-key
VITE_EXPORT_API_BASE_URL=https://your-cgm-export-backend.example.com
```

If Supabase variables are missing, the app shows a clear data-status warning and uses local recovery data only. This is not suitable for production office use.

## Export Backend

Professional client documents are generated by the Python FastAPI backend in `backend/`.

Supported endpoints:

- `GET /health`
- `POST /exports/quotation`
- `POST /exports/invoice`
- `POST /exports/receipt`
- `POST /exports/client-statement`
- `POST /exports/excel`
- compatibility: `POST /api/export/pdf`, `POST /api/export/excel`, `POST /export/pdf`, `POST /export/excel`

Known client document PDFs must use the backend. The frontend no longer silently downloads raw field/value PDFs for quotations, invoices, receipts, or client statements if the backend is unavailable.

## Data Safety

- Do not renumber existing records.
- Do not delete audit logs.
- Download a backup before restore, settings changes, or month-end work.
- Restore validates backup shape, downloads a pre-restore safety backup, and records the action in the audit log.

## Verification

```bash
npm run build
cd backend
python -m pytest
```

Backend tests require `fastapi`, `httpx`, `pytest`, `pydantic-settings`, `reportlab`, and `openpyxl` from `backend/requirements.txt`.
